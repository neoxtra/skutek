SkuTek Streaming Setup And Description
============================================

The DDC HAL Software has several functions that are relatively simple to work with for sending wave data between systems. 
There are currently two packet types available, a GRETINA packet that is the Wave0 waveform GRETINA format for SkuTek and 
a GRETA packet that is a special wave trace described in this chapter conforming to the Gamma Ray Energy Tracking Array format.

There is one thing to note, that the maximum number of samples that can be sent over the UDP streaming system is 512, as opposed to a "full trace" of 4096 samples.
It is important to note that sequence numbers are defined by the DDC_stream_context, an individual context will have a initial sequence number and will increment that for all packet types that have sequence numbers.


Greta Packet Structure
----------------------

The GRETA packet sent by the Vireo is a UDP packet to a given address, because it is a UDP packet, there are no gaurantees on ordering or arrival at all, without specialized setups.
SkuTek Instrumentation sells a Data Collection system for ensuring ordering and capture more reliably than self-made systems.


GRETA Packet Routing Header
^^^^^^^^^^^^^^^^^^^^^^^^^^^
+---------------------+--------+--------------------------------------+
| Word                | Type   | Info                                 |
+=====================+========+======================================+
| **Version**         | UInt8  | Protocol Version                     |
+---------------------+--------+--------------------------------------+
| **Flags**           | Int16  | Flags                                |
+---------------------+--------+--------------------------------------+
| **Type**            | UInt8  | Type of Header                       |
+---------------------+--------+--------------------------------------+
| **Subtype**         | UInt8  | Subtype of Header                    |
+---------------------+--------+--------------------------------------+
| **Length**          | UInt16 | Length of Payload                    |
+---------------------+--------+--------------------------------------+
| **Sequence Number** | UInt16 | The ordering number of the events    |
+---------------------+--------+--------------------------------------+
| **Timestamp**       | Int64  | Timestamp of the event               |
+---------------------+--------+--------------------------------------+
| **Checksum**        | Int64  | Checksum, currently unused by SkuTek |
+---------------------+--------+--------------------------------------+
The C version of this code is as follows:

.. code:: C
   
   struct greta_routing_header
   {
      uint8_t version; // protocol version
      uint8_t flags;
      // debugging
      uint8_t type;
      // Main packet type - use the module number
      uint8_t subtype; // message sub-type - use the channel number

      uint16_t length;   // length of payload
      uint16_t seqnum;   // sequence number. Incremented for each event
      int64_t timestamp; // generated by trigger
      int64_t checksum;  
   } __attribute__((__packed__, __aligned__(4)));

SkuTek GRETA Wave Subheader
^^^^^^^^^^^^^^^^^^^^^^^^^^^
+-----------------------------+-------------+--------------------------------------------+
| **Subheader Version**       | Int8        | The version of the wave subheader          |
+-----------------------------+-------------+--------------------------------------------+
| **Trig Count**              | UInt8       | The trigger count                          |
+-----------------------------+-------------+--------------------------------------------+
| **Triggered **              | UInt8       | Whether the channel was triggered, boolean |
+-----------------------------+-------------+--------------------------------------------+
| **Reserved**                | Int8        | Reserved                                   |
+-----------------------------+-------------+--------------------------------------------+
| **Trigger Height**          | Int16       | The height of the triggered event          |
+-----------------------------+-------------+--------------------------------------------+
| **Module Number**           | UInt16      | The Module Number of the Vireo             |
+-----------------------------+-------------+--------------------------------------------+
| **Channel**                 | UInt16      | Channel of the Vireo Triggered             |
+-----------------------------+-------------+--------------------------------------------+
| **Start Location**          | UInt16      | Start Location of the Event                |
+-----------------------------+-------------+--------------------------------------------+
| **QDC Baseline**            | Int32       | QDC Integration Baseline                   |
+-----------------------------+-------------+--------------------------------------------+
| **QDC Integration -- Fast** | Int32       | QDC Integration Fast                       |
+-----------------------------+-------------+--------------------------------------------+
| **QDC Integration -- Slow** | Int32       | QDC Integration Slow                       |
+-----------------------------+-------------+--------------------------------------------+
| **QDC Integration -- Tail** | Int32       | QDC Integration Tail                       |
+-----------------------------+-------------+--------------------------------------------+
| **Size**                    | UInt32      | Number of Elements being Sent              |
+-----------------------------+-------------+--------------------------------------------+
| Reserved                    | UInt8 \* 64 | Reserved for future use                    |
+-----------------------------+-------------+--------------------------------------------+


.. code-block:: C
   
   struct skutek_greta_wave0
   {
      // Version of this subheader
      int8_t subheader_version;
      // Trigger count
      uint8_t trig_count;
      // triggered boolean, done as an int8
      int8_t triggered;
      // Reserved, not for use
      int8_t reserved_0;
      // The height the trigger was triggered
      int16_t trig_height;
      // Pulse height
      int16_t pulse_height;
      // The module number
      uint16_t module_number;
      // Channel number that was triggered
      uint16_t channel;
      // Start location of the pulse
      int16_t start_location;
      // 2nd reserved slot.
      int16_t reserved_1;
      // While the Vireo itself does not have QDC, because this is a common format between Vireo and Kingfisher, include
      // it.
      int32_t qdc_base_sum; // QDC integration - baseline
      int32_t qdc_fast_sum; // QDC integration - fast
      int32_t qdc_slow_sum; // QDC integration - slow
      int32_t qdc_tail_sum; // QDC integration - tail
      // Number of samples following this subheader
      uint32_t size;
      uint8_t reserved_3[64];
      // This is followed by what would be a "natural" flexible array member, allocated immediately after this structure
   } __attribute__((__packed__, aligned(4)));

Packet As Sent Over the Wire
^^^^^^^^^^^^^^^^^^^^^^^^^^^^
C code:

.. code-block:: C
   
   struct
   {
      struct greta_routing_header header;
      struct skutek_greta_wave0 wave;
      int16_t data[];
   } __attribute__((__packed__));

The packet shown above is the packet as sent over the wire, and is not used anywhere in the DDC Streaming Application publically, 
however it is important to note for the wave packet format as this is how it is literally sent over the wire.
